# コンポーネント設計

## 方針

- 1ファイル1コンポーネント
- UIとロジックを分離する。ロジックはカスタムhookまたは `lib/` に置く
- スタイリングはTailwind CSSのユーティリティクラスを使う。CSS Modulesは使わない

## コンポーネントの分類

- **ページコンポーネント** (`app/**/page.tsx`): データ取得とレイアウト構成のみ
- **機能コンポーネント** (`components/`): 特定の機能を持つUI部品
- **共通コンポーネント** (`components/ui/`): ボタン、カードなど汎用UI（必要になったら作る）
